# SourceCraft Code Assistant (Yandex Code Assistant)

## О сервисе

SourceCraft Code Assistant — AI-помощник для разработчика внутри платформы SourceCraft и в виде плагинов для IDE. На текущей официальной странице сервис описан как ассистент с агентским режимом, встроенным чатом, code review, контекстными подсказками и поддержкой популярных редакторов, включая VS Code и IDE от JetBrains.

Если вы привыкли к старому названию, это тот же продукт: в FAQ SourceCraft прямо сказано, что `Yandex Code Assistant` стал частью платформы SourceCraft и был переименован в `SourceCraft Code Assistant` без потери функциональности.

Для LeanToken особенно важно другое: актуальная документация SourceCraft уже описывает профили моделей и работу с внешними AI-провайдерами. Это делает сценарий BYOK через `https://leantoken.tech` естественным, а не обходным.

## Где SourceCraft Code Assistant особенно полезен

- агентная работа над задачами на естественном языке прямо в IDE
- разбор репозитория и объяснение структуры проекта через встроенный чат
- code review и быстрые AI-подсказки без переключения в отдельный сервис
- `Next Edit Action` и inline completion во время редактирования кода
- генерация документации, тестов и рутинных правок по контексту проекта
- подключение MCP-серверов и разных LLM под разные режимы работы

## Что важно знать перед настройкой BYOK

Перед настройкой LeanToken полезно понимать два базовых механизма SourceCraft Code Assistant.

### Профили моделей

По официальной документации в Visual Studio Code ассистент использует `configuration profiles`. Один профиль может хранить:

- AI provider
- API key
- модель
- temperature и другие model settings
- rate limit
- дополнительные provider-specific параметры

Это удобно для BYOK, потому что можно не менять глобальные настройки каждый раз, а завести отдельный профиль, например `LeanToken`.

### Режимы работы

В документации SourceCraft для режима агентов описаны встроенные режимы:

- `Code`
- `Ask`
- `Architect`
- `Debug`
- `Orchestrator`

Для каждого режима можно помнить отдельный профиль и отдельную модель. Практически это значит, что через LeanToken можно:

- держать более сильную модель для `Code` или `Architect`
- использовать более быструю или дешевую модель для `Ask`
- тестировать разные профили, не ломая базовую конфигурацию

## Что понадобится

- Visual Studio Code с установленным плагином SourceCraft Code Assistant
- аккаунт на `https://leantoken.tech`
- API key LeanToken
- OpenAI-compatible endpoint `https://api.leantoken.tech/v1`
- model ID, доступный в вашем аккаунте LeanToken

Ниже сценарий описан именно для VS Code. Это самый надежный стартовый путь для первой версии страницы, потому что текущая официальная документация по профилям моделей опубликована для Visual Studio Code, и сохраненный скриншот тоже относится к этой версии интерфейса.

## BYOK на leantoken.tech

В контексте SourceCraft Code Assistant Bring Your Own Key означает, что вы не ограничиваетесь только встроенным профилем с моделями Яндекса, а создаете собственный профиль для внешнего OpenAI-compatible провайдера.

Для LeanToken это выглядит так:

1. вы получаете свой API key в LeanToken
2. в Code Assistant создаете отдельный профиль
3. выбираете `OpenAI Compatible`
4. указываете `https://api.leantoken.tech/v1`
5. вставляете свой LeanToken API key
6. выбираете нужную модель
7. при необходимости привязываете профиль к нужному режиму работы

Такой подход удобен, когда вы хотите сами контролировать:

- модель под конкретный сценарий работы
- стоимость и квоты
- переключение между несколькими профилями
- границу между встроенными и внешними моделями

## Как получить API key на leantoken.tech

Сначала получите key и endpoint в LeanToken.

![Публичная страница LeanToken с Sign up, Log in и Dashboard](../../images/leantoken/leantoken.tech.1.landing.png)

1. Откройте `https://leantoken.tech`.
2. Нажмите `Sign up`, если у вас еще нет аккаунта, `Log in`, если аккаунт уже есть, или `Dashboard`, если вы уже авторизованы.

![Панель LeanToken с API endpoint и быстрыми ссылками](../../images/leantoken/leantoken.tech.2.dashboard.png)

3. После входа откройте dashboard или обзорную страницу API.
4. Скопируйте API endpoint. На текущих скриншотах показан `https://api.leantoken.tech/v1`.
5. Перейдите в раздел управления ключами через `Manage keys` или `API Keys`.

![Страница API Keys в LeanToken с действием Create key](../../images/leantoken/leantoken.tech.3.key-managment.png)

6. Нажмите `Create key`.
7. Скопируйте созданный API key.
8. Вернитесь в настройки SourceCraft Code Assistant и вставьте key в поле `OpenAI API Key`.
9. Храните key как секрет. Если он попал в git, логи, скриншоты или историю терминала, перевыпустите его.

## Как подключить LeanToken в SourceCraft Code Assistant

На сохраненном скриншоте ниже все еще видно старое имя `Yandex Code Assistant Settings`, но сам сценарий настройки остается актуальным: продукт переименован в SourceCraft Code Assistant, а экран профилей моделей по-прежнему подходит для BYOK-потока.

![Экран настроек SourceCraft Code Assistant с профилем Custom, провайдером OpenAI Compatible, LeanToken Base URL и выбранной моделью](../../images/yandex-code-assistant/yandex-code-assistant.png)

### Пошаговая настройка

1. Откройте чат SourceCraft Code Assistant в Visual Studio Code.
2. В верхней панели чата откройте `Settings`.
3. В разделе `Models` нажмите `+` рядом с селектором `Profile`.
4. Создайте новый профиль. Практичное имя: `LeanToken`.
5. В поле `API Provider` выберите `OpenAI Compatible`.
6. В поле `Base URL` укажите `https://api.leantoken.tech/v1`.
7. В поле `OpenAI API Key` вставьте ваш LeanToken API key.
8. В поле `Model` выберите модель, доступную в вашем аккаунте LeanToken.
9. Если пока не уверены в дополнительных настройках, оставьте значения по умолчанию для `temperature`, `rate limit` и других provider-specific опций.
10. Сохраните профиль и вернитесь в чат.

Минимальная рабочая конфигурация выглядит так:

```text
Profile: LeanToken
API Provider: OpenAI Compatible
Base URL: https://api.leantoken.tech/v1
OpenAI API Key: <ваш_LeanToken_API_key>
Model: <ваш_model_id>
```

По официальной документации ключи в этом потоке хранятся в `VS Code Secret Storage`, а не как открытый текст в интерфейсе.

## Как использовать профили вместе с режимами

Если вы уже работаете в агентном режиме, после создания профиля удобно привязать его к конкретным сценариям:

- `Code` для основной реализации и изменения файлов
- `Ask` для объяснений без правок проекта
- `Architect` для планирования и проектирования
- `Debug` для диагностики проблем

Практическая схема для первого запуска:

- один профиль LeanToken для всех режимов на старте
- потом отдельный профиль под дорогую сильную модель для `Code`
- и отдельный профиль под более быструю модель для `Ask`

## Первый smoke test после настройки

После сохранения профиля не начинайте сразу с большой задачи. Сначала проверьте базовый контур.

1. Отправьте простой запрос, например: `Прочитай README и кратко объясни, как запускается проект`.
2. Попросите ассистента показать релевантные файлы или короткий план изменений.
3. Если вы уже используете режимы, проверьте, что профиль корректно подхватывается в `Code` или `Ask`.
4. После этого переходите к маленькой практической задаче: обновить текст, сгенерировать тест или объяснить конкретный модуль.

## Что стоит проверить, если ответ не приходит

- Убедитесь, что в `Base URL` указан именно `https://api.leantoken.tech/v1`.
- Проверьте, что API key вставлен полностью и без лишних пробелов.
- Проверьте, что выбранный model ID действительно доступен в вашем аккаунте LeanToken.
- Если профиль был создан давно, откройте настройки заново и перепроверьте список моделей.
- Если нужна особенно строгая политика данных, отдельно изучите актуальные настройки SourceCraft, связанные с передачей данных в модели Яндекса и внешние провайдеры.
